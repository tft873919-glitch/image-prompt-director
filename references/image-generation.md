# 锁版生图与成果管理

## 进入条件

只有同时满足以下条件才生图：

1. 用户明确要求直接生成。
2. 项目记录状态为“已锁版”或用户当轮明确锁定某个 Prompt 版本。
3. 对应的单画面运行 Prompt（render-prompt）通过校验。
4. 当前使用的参考图及角色没有歧义。

讨论、比较、反推和 Prompt 编写阶段不要自动生成预览图。

## 输入组装

- 完整读取锁定的 `<项目名>_vX.Y.Z.md` 单画面运行 Prompt，原样作为主要生成规格；不要在调用生图时静默改 Prompt。
- 如果用户只锁定了 `<项目名>_风格规格_vX.Y.Z.md`，先从风格规格中提取当前画面所需内容，另存为独立、完整的 render-prompt 并请用户锁定；不要把包含模板、示例或变体的整份风格规格直接送去生图。
- 从项目记录的“参考图分工”读取当前仍有效的图片，把每张图按 identity、visual-style、visual-system、composition、material/lighting 或 edit-target 标注。
- 使用所有与锁定共识相关的参考图，但不要把旧出图、被否定参考、聊天截图或无关附件混入。
- 如果所有目标图片都有本地路径，使用 `referenced_image_paths`。
- 只要有目标图片没有本地路径，就用最小的 `num_last_images_to_include` 覆盖所有目标图片；不要同时传两种机制。
- 本地编辑目标先用 `view_image` 检查；生成型参考不等于编辑目标。

## 生成与保存

默认使用 `$imagegen` 的内置生成模式。生成后检查主体、身份锚点、构图/图层、准确文字、材质、色彩和关键限制。

项目成果必须从默认 `$CODEX_HOME/generated_images/...` 位置复制或移动到：

```text
outputs/<中文项目名>/<Prompt文件主名>.<实际扩展名>
```

例如：

```text
prompt/冬夜敲门/冬夜敲门_v1.1.0.md
outputs/冬夜敲门/冬夜敲门_v1.1.0.png
```

- 保留实际文件格式，不把 WebP/JPEG 只改后缀伪装成 PNG。
- 不覆盖已有成果。同 Prompt 再次生成时用 `_02`、`_03`。
- 多张成果按 `_01`、`_02` 顺序保存。
- Prompt 有任何实质修改时先升级版本，再生成同版本主名的成果。
- 保存成功后更新项目记录的当前成果、生成时间、使用版本和链接。

## 迭代边界

- 用户只要求“再抽一张”，Prompt 不变，新增成果序号。
- 用户反馈要求改变画面，先回到迭代阶段，升级 Prompt 后再生图。
- 用户要求编辑已有成果时，将该成果设为 edit-target，并把其他参考图角色保持清楚。
- 不为了修图方便静默放弃身份参考、文字准确性或已锁定构图。

## 最终响应

遵循 `$imagegen` 的最终响应约束：生成完成后展示生成结果，不追加下载说明、总结或追问。项目文件保存和记录更新应在同一工作流中完成。
